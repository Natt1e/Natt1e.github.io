基于Ubuntu18.04配置snort+barnyard2+base入侵检测系统

我们一共需要: Apache , PHP, MySQL, Snort, Barnyard2, BASE.

Ubuntu版本为18.04 LTS.

安装Apache2 Web服务器

sudo apt-get install apache2

安装PHP

因为环境比较古早，需要PHP5及以下的版本。但Ubuntu18.04自带了PHP7，所以需要新装一个PHP5，并且切换至PHP5

安装PHP5：

sudo add-apt-repository ppa:ondrej/php
sudo apt-get update
sudo apt-get install php5.6

将版本切换至5：

  1. 先禁用PHP 7，可以用php -v先查看当前的php版本。
sudo a2dismod php7.4

输出：Module php7.4 already disabled

如果提示没有mod就先安装一个：sudo apt-get install libapache2-mod-php7.4

  1. 启用PHP5
sudo a2enmod php5.6

看到最后一行输出Module php5.6 already enabled即为成功，除此之外应该还会有一些其他的信息。

  1. 重启apache
sudo service apache2 restart
  1. 查看当前PHP版本
php -v

此时应该成功切换至PHP5，如果未切换成功，则等待几分钟，再进行查看。

安装mysql

直接进行安装即可

sudo apt-get install mysql-server
sudo apt-get install mysql-client

安装完以上之后，还需安装一些其他依赖项

apache2 php5模块

sudo apt-get install libapache2-mod-php5.6

php5 mysql模块

sudo apt-get install php5.6-mysql

安装snort

snort是一个入侵检测系统，在1998年用C语言开发，已经是非常古老的产物了。

  1. 先查看一下本机的接口：（之后会用到）
ifconfig

一般来讲都会有ens33和lo两个接口。如果提示没有net-tools就安装一个。

  1. 安装snort，直接安装即可
sudo apt-get install snort

在安装过程中，可能会弹出窗口让选择Snort监听的接口，默认为eth0，但可能本机根本就没有eth0这个接口。此时填ifconfig查询出来的接口即可，如ens33.

  1. 查询Snort版本，观察是否安装成功
snort -V

  1. 编辑配置文件

snort的配置文件在/etc/snort/snort.conf

sudo vim /etc/snort/snort.conf

修改内容如下(若内容一样则无需修改)：

（vim跳转至xx行，可直接:xxx回车）

115行左右：

var RULE_PATH /etc/snort/rules
var SO_RULE_PATH /etc/snort/so_rules
var PREPROC_RULE_PATH /etc/snort/preproc_rules

目的为指定规则文件存放的路径，一般不用修改。

536行左右：

output unified2: filename snort.log, limit 128, mpls_event_types, vlan_event_types

目的为配置日志输出选项。

output unified2: 设置日志输出格式为“unified2”

filename snort.log: 将日志写入名为“snort.log”的文件

limit 128: 设置日志文件的最大大小为128MB

mpls_event_types: 启用对MPLS（多协议标签交换）事件的记录

vlan_event_types: 启用对VLAN（虚拟局域网）事件的记录

**若显示该文件为只读文件，可强制修改，命令为:w !sudo tee %**，该指令以后也会用到。

测试snort

接下来，我们来测试一下snort，体会一下它的功能：

  1. 启动snort
sudo service snort restart
  1. 修改规则
sudo vim /etc/snort/rules/local.rules //这个是不是似曾相识？这是存放自定义规则的文件

向其中加入：

alert icmp any any -> $HOME_NET any (msg:"ICMP Test NOW!!!"; classtype:not-suspicious; sid:1000001; rev:1;)
alert tcp any any -> $HOME_NET 80 (msg:"HTTP Test NOW!!!"; classtype:not-suspicious; sid:1000002;  rev:1;)

这两条规则的含义是无论什么发往本机的ICMP和HTTP报文都会发出警告。

  1. 加入规则
sudo vim /etc/snort/snort.conf

在大概573行：将除了local.rules之外的规则文件都去掉，即那些include，目的是为了方便测试。

最终是这样：

# site specific rules
include $RULE_PATH/local.rules
  1. 启动测试
sudo snort -T -c /etc/snort/snort.conf

如果看到输出是这样的：

...
...
Snort successfully validated the configuration!
Snort exiting

并且还能在一堆信息里找到：

配置Mysql

我们配置Mysql的目的是创建一个snort用户，和两个数据库snort和archive，以便信息可以存储到数据库中。

  1. 先尝试连接数据库：
mysql -u root -p

-u root ： 用root身份

-p ：需要密码

提示输入密码，直接按回车，如果报错，则进行下面的环节，如果不报错可跳过2-3步。

  1. 设置允许无密码登录，重启mysql

修改文件：

sudo vim /etc/mysql/mysql.conf.d/mysqld.cnf

在[mysqld]中添加skip-grant-tables,直接添加即可

重启mysql：

service mysql restart

再连接mysql：

mysql -u root -p

此时直接按回车便可登录。

  1. 设置密码，取消无需密码登录

别忘了分号

use mysql;
flush privileges;
UPDATE user SET authentication_string="" WHERE user="root";//先把root密码置为空
flush privileges;
alter user 'root'@'localhost' identified with mysql_native_password by '你的密码'; // 重置密码
exit; // 退出

再去掉之前加的skip-grant-tables

sudo vim /etc/mysql/mysql.conf.d/mysqld.cnf

重启mysql

service mysql restart
  1. 配置用户以及数据库
mysql -u root -p

输入重置后的密码

create database snort;
create database archive;// 创建snort 和 archive数据库
create user snort@localhost; // 创建snort用户
grant usage on snort.* to snort@localhost; // 授予权限，只能从本地连接服务器
grant usage on archive.* to snort@localhost;// 同上
set password for snort@localhost=PASSWORD('密码'); // 设置密码，这个密码要记住，以后要用
grant all privileges on snort.* to snort@localhost; // 授予对数据库的所有操作
grant all privileges on archive.* to snort@localhost;// 如插入、删除等等..
flush privileges;
exit

安装barnyard2

  1. 首先安装一些依赖项
sudo apt-get install build-essential libpcap-dev libmysqlclient-dev libtool libssl-dev libnetfilter queue-dev libpcre3-dev autoconf automake bison flex libdumbnet-dev

执行失败也没关系，之后缺什么按什么即可

  1. 安装barnyard2
cd ~
wget https://github.com/firnsy/barnyard2/archive/v2- 1.13.tar.gz -O barnyard2-2-1.13.tar.gz
tar zxvf barnyard2-2-1.13.tar.gz
cd barnyard2-2-1.13
autoreconf -fvi -I ./m4
./configure --with-mysql --with-mysql libraries=/usr/lib/x86_64-linux-gnu --with-mysql-includes=/usr/include/
make
sudo make install
sudo cp /usr/local/etc/barnyard2.conf /etc/snort
sudo mkdir /var/log/barnyard2
touch /var/log/snort/barnyard2.waldo
sudo chown snort.snort /var/log/barnyard2
sudo cp schemas/create_mysql /usr/src   //这个之后要用，在barnyard2-2-1.13目录下，我一开始没找到。。
  1. 编辑配置

44行附近，修改为：

config logdir:/var/log/barnyard2

56行附近，修改为：

config hostname: localhost
config interface: ens33 //是不是感觉似曾相识

131行附近，修改为：

config waldo_file:/var/log/snort/barnyard2.waldo

227行附近，修改为：

output alert_fast

348行附近，去掉注释，修改为：

output database: log, mysql, user=snort password=123456 dbname=snort host=localhost

注意：这里的user和password填刚刚在数据库创建的用户及其密码

还记得之前说过sudo cp schemas/create_mysql /usr/src之后要用吗，就是现在要用一下，这是一个建表的脚本。

mysql -u root -p

输入密码登录数据库

use snort;
source /usr/src/create_mysql;
show tables;                                                 
exit

  1. 各种安装

安装相关插件

sudo apt-get install libphp-adodb

修改php.ini:

sudo vi /etc/php/5.6/apache2/php.ini

在449行附件，改为：

error_reporting = E_ALL & ~E_NOTICE

这个要改对哦，我一开始没改对，找了好久。。。

修改apache2.conf:

sudo vim /etc/apache2/apache2.conf

加入以下内容：（178行左右）

<Directory /var/www/html/base>
    AllowOverride All
    Require all granted
</Directory>

重启Apache2：

sudo service apache2 restart

安装BASE依赖：

sudo apt-get install php-pear

sudo apt-get install libwww-perl                                                       

sudo apt-get install php5.6-gd

sudo apt-get install php5.6-xml

sudo pear config-set preferred_state alpha

sudo pear channel-update pear.php.net

sudo pear install --alldeps Image_Color Image_Canvas Image_Graph
